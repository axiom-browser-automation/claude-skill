When an automation fails or behaves unexpectedly, the pages in this section help you find out why. Use the [debugger and Display a message step](/docs/no-code-tool/troubleshooting/how-to-debug) to inspect what's happening during a run, and the [error handling guide](/docs/no-code-tool/troubleshooting/error-handling) to keep an automation running through expected failures.

## Troubleshooting guides
***

Start with the debugger when you need to see what's happening during a run. Move on to error handling once you understand the failure and want the automation to react to it gracefully. The common errors reference covers specific error messages, what they mean, and how to fix each one.
