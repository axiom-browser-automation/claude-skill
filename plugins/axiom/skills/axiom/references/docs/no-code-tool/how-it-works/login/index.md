There are several ways to keep an automation signed in to the web apps it visits. The simplest is to share your browser [session](/docs/no-code-tool/how-it-works/login/sessions). For organisations that need stricter isolation, a [secure login](/docs/no-code-tool/how-it-works/login/secure) setup keeps everything on a controlled PC. Don't store sensitive passwords directly in step fields, even though all step data is encrypted.
