Error handling settings change how an automation reacts to errors and edge cases during a run. Use [continue on error](/docs/no-code-tool/reference/settings/error-handling/continue-on-error) to keep going past failures, [iframe support](/docs/no-code-tool/reference/settings/error-handling/iframes) to interact with embedded frames, and [page monitoring](/docs/no-code-tool/reference/settings/error-handling/page-monitoring) to control how axiom.ai detects when a page has finished loading.
