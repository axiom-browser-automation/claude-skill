## What to use the Display a message step for
***

The Display a message step is mainly utilised for debugging Axioms. It is similar to a 'Console Log' in programming, but is shown on the screen during runtime. This step is helpful in outputting data from previous steps to better comprehend the current situation. Additionally, it can be utilised to verify if any conditional logic is functioning as anticipated. For instance, when using the [Conditional Jump Step](/docs/no-code-tool/reference/steps/Conditionall-jump-step), you can trigger a message when the conditions are either true or false.

You can use this step to:

- Display data output by steps
- Check conditional logic is working as expected
- Flag the step your automation is on

## Configuration
***

### Message

Type in the message you would like to display. You can also display data from other steps by clicking 'Insert Data.' You can add multiple tokens.
